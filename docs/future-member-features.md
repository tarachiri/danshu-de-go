# 会員登録・Myカレンダー・例会登録機能（将来実装・設計メモ）

> 断酒会の例会情報を、一人一人に合わせた形で届けるための土台となる機能群。
> 現時点では未実装。エンティティ設計の初期整理として記録する。

## この3機能の関係

断酒でGO!!は現在、断酒会（全日本断酒連盟加盟）の例会・施設情報を「誰でも見られる地図」として提供している。この3機能は、そこに「利用者ごとのカスタマイズ」を積み重ねる拡張群であり、依存関係を持つ。

```
【会員登録機能】（土台）
   ↓ を前提に
【Myカレンダー機能】（一般利用者向け・軽い操作）
   ↓ とは別に、より厳密な認証を前提とする
【例会登録機能】（例会の所属者向け・重い操作）
```

会員登録は、アルコール依存症からの回復を目指す本人やその家族が、断酒会の例会・ミーティング情報に継続的にアクセスしやすくするための入口になる。

## 機能1：会員登録機能

断酒でGO!!アプリへの会員登録（メンバー登録、名称は検討中）により、利用者一人一人に合わせたサービス提供が可能になる。以下の2機能（Myカレンダー・例会登録）の前提となる基盤機能。

### 想定する情報

- 表示名
- 連絡先（メールアドレス等）
- 会員種別（一般会員 / 例会管理会員）※詳細は**TBD**（下記参照）
- 所属団体・グループ（エリア）

### 現時点でのTBD（未確定事項）

エンティティ設計段階で、以下2点は実態確認後に判断する方針とし、あえて先送りしている。

1. **会員種別（一般会員／例会管理会員）の扱い**
   - 案A：1つの会員テーブルに種別フラグを持たせる（シンプル）
   - 案B：一般会員と例会管理会員を別テーブルに分ける（責務分離）

## 機能2：Myカレンダー機能

会員登録済みの利用者が、自分の所属団体・お気に入り例会を登録することで、表示される例会スケジュールをカスタマイズできる機能。

### 要件

- **お気に入り例会登録**：気になる例会・自分が通う例会を個別に登録
- **所属団体の反映**：所属している断酒会の例会を自動的にカレンダーへ反映
- **PDF印刷対応**：紙で持ち歩きたい利用者（特に高齢の利用者）への配慮
- **文字サイズ**：大きめ表示に対応し、例会名が途中で見切れないようにする
- **表示モード選択**：
  - 例会名のみ表示
  - 開始時間＋例会名を表示

これは、PWA（Web Push）が高齢利用者層には「未経験の越境操作」という経験値の壁になりやすいという既存の知見とも関係する。カレンダーのUIも同様に、迷わず・見やすく操作できることを優先する。

### 将来のスコアリング構想

所属団体・お気に入り例会に加え、以下のような多基準スコアリングによる例会の推薦・並び替えも構想中（SQLite内での実装を想定）。

- 日程の近さ
- 最寄り駅からの距離
- 自宅からの距離

これらを重み付け合算し、利用者にとって「行きやすい例会」を優先表示する仕組み。

## 機能3：例会登録機能

会員登録機能よりもさらに厳密な本人確認・所属確認を前提とする機能。所属団体、所属グループ（エリア）、所属する断酒会の例会を登録した会員が、その例会情報の登録・変更・削除を行えるようにする。

### なぜ厳密さが必要か

例会情報（開催場所・時間・連絡先等）は、断酒会に足を運ぼうとしている本人が最初に頼る情報になる。誤った情報の登録・改ざんは、勇気を出して例会に向かった人を裏切ることになりかねないため、一般のMyカレンダー機能（お気に入り登録という軽い操作）とは明確に分離して扱う。

### 想定する情報

- 所属団体
- 所属グループ（エリア）
- 管理対象の例会（既存の `meetings` データとの紐付け）
- 本人確認・検証状況
