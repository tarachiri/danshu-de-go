# インシデント記録：断かも「今日/明日/今から」の週次判定バグ

作成: 2026-07-09 (ふーちゃん / Claude Code, soi上)
発端: まじまじさんからの質問「断かも　今日の例会　青梅例会　八王子例会　立川第二　
なぜそう答える？どれも今日じゃないはずだが何が原因だ？」

## 問題

`chat.html`で「今日の例会を探す」を選ぶと、実際にはその日に開催されない例会が
混ざって表示されていた。

例：2026-07-09（7月の**第2木曜**）に「今日の例会」を聞くと、
青梅例会（第1木曜）・八王子例会（第4・5木曜）・立川第2例会（第3木曜）が
ヒットしていた。いずれも「木曜日」という曜日だけが一致しており、
「第何木曜か」は無視されていた。

### 原因

`searchVenues()`（[chat.html:307-312](../../chat.html#L307)付近）の
`isToday` / `isTomorrow` / `isNow` 判定が、`meeting.day_of_week`
（曜日）だけを見て一致判定しており、`meeting.week_of_month`
（第何曜日か）を一切見ていなかった。

```js
} else if (isToday) {
  if (meetingDay === todayFull || meetingDay === todayShort) match = true;
}
```

## 対応内容

`venues.json`の各例会に元々ある`next_date`フィールド（`generate_map_v6.py`が
`day_of_week`・`week_of_month`に加えて`schedule_exceptions`（中止・変更）も
考慮して計算済みの「本当の次回開催日」）と、クライアント側で取得した
今日・明日の日付文字列を直接比較する方式に変更した。

```js
} else if (isToday) {
  if (meeting.next_date === todayStr) match = true;
}
```

「曜日で探す」（第何曜日かを問わず、木曜日全般の例会一覧を見せる機能）は
仕様として意図的に曜日一致のみのままにしている。

### 検証

- プレビューでUIを実際に操作し、2026-07-09（第2木曜）に「今日の例会」を
  聞いたところ、鈴鹿・伊勢・志摩（いずれも第2木曜を含む周期）のみが
  返るようになったことを確認。青梅・八王子・立川第2例会は出なくなった。

### コミット

- [f183fda](https://github.com/tarachiri/danshu-de-go/commit/f183fda) —
  `chat.html`のnext_date基準への修正（同コミットに、無関係な既存の
  未コミット変更＝問い合わせ送信用`callInquiry()`も含まれている）

## 議題（要検討・要確認事項）

1. **`next_date`の鮮度**：`next_date`は毎日5:00の`generate_map_v6.py`実行時点で
   計算される。その日のうちに`schedule_exceptions`へ新しい中止・変更が
   追加された場合、次のcron実行（翌5:00）まで`next_date`には反映されない。
   実運用上「今日中止になった」お知らせがどれくらいの頻度で当日に
   追加されるか次第では、断かもの回答が古い情報のままになるリスクがある。
2. **同種のロジックの他箇所への横展開**：今回直したのはchat.htmlの
   「今日/明日/今から」のみ。地図側（index.html/app.js）や他の
   「次回開催」表示ロジックが同じ曜日オンリー判定をしていないか、
   一度棚卸しが必要かもしれない。
3. **[2026-07-09_venue_matching_generic_word_bug.md](2026-07-09_venue_matching_generic_word_bug.md)との関係**：
   同日に見つかった別バグ（venue_idの正規表現誤マッチ）とは無関係だが、
   どちらも「表示されている例会が実際の開催実態とズレる」という
   同種のユーザー影響のため、あわせて参照されたい。

## 2026-07-09 追加対応：gen本番API側の同種バグ

### 再発内容

Web側 `chat.html` は `next_date` 基準に修正済みだったが、断かも本体
（gen: `/Users/mini2014/danshu-chat/main.py`）の `search_venues()` が
まだ `day_of_week` だけで「今日」「明日」「今から」を絞り込んでいた。

そのため、LINE/Web API経由の断かもでは、2026-07-09（第2木曜）に
青梅例会（第1木曜）、立川第2例会（第3木曜）、八王子例会（第4・5木曜）が
今日候補として扱われる余地が残っていた。

### 変更点

- gen本番 `main.py` を修正。
- 修正前バックアップを `main.py.bak-20260709-today-next-date` として作成。
- 「今日」「明日」「今から」は `meeting.next_date` と対象日付を比較する方式へ変更。
- 「木曜日」などの曜日検索は、従来通り `day_of_week` による曜日一覧検索として維持。
- Claudeへ渡す関連例会情報に `next_date` と `recurrence` を含めるようにし、
  「今日ではないが次回はいつか」を誤解しにくくした。
- `launchctl kickstart -k gui/$(id -u)/com.danshu.uvicorn` でFastAPIを再起動。

### 検証

gen上でHTTP APIへ直接POSTし、以下を確認した。

- `今日 青梅例会` → 今日開催ではない、次回は 2026-08-06 と返答。
- `今日 立川第二` → 今日開催ではない、次回は 2026-07-16 と返答。
- `今日 八王子例会` → 今日開催ではない、次回は 2026-07-23 と返答。

### 注意

同じ「今日の例会」でも、静的Webチャット `chat.html` と gen上の
FastAPI/LINE断かもは別実装である。今後この種の日時判定を直す時は、
必ず両方の検索経路を確認する。

## 2026-07-09 棚卸し結果：地図側（index.html/app.js）は対象外

作成: ふーちゃん / Claude Code, soi上

議題2「同種のロジックの他箇所への横展開」を受けて `app.js` を確認した。

**結論：地図側にこのバグは無い。** `app.js` の `getDateLabel()`
（[app.js:80-86](../../app.js#L80)）は元々 `next_date`
（実際の日付文字列）と `TODAY`/`TOMORROW`/`DAY_AFTER` を直接比較する
方式で実装されており、`day_of_week`・`week_of_month` はピンの
today/tomorrow/dayafterバッジ判定に一切使われていない
（`grep`で該当箇所なし）。

つまり今回の「曜日だけ判定」バグは `chat.html` と gen `main.py` の
2箇所に限定されており、地図のピン表示・バッジは最初から正確だった。
横展開の懸念は解消。
