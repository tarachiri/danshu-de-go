# 2026-07-07 断酒でGO!!GO!! 初期設計メモ

## 今日の課題

- 例会情報を登録する別アプリ「断酒でGO!!GO!!」を、断酒でGO本体とどう分けるか。
- 投稿できるデータ形式をどこまで広げるか。
- GitHub Pages、自宅サーバー、独自ドメイン、Androidアプリ分割の方針を決める。

## 入力データ形式

初期対応候補:

- PDF
- Excel
- Word
- テキストデータ

追加候補:

- CSV / TSV: 表形式の投稿に向いており、変換精度が高い。
- 画像: チラシや紙の予定表をスマホ撮影して送れる。OCRが必要。
- iCalendar / ICS: 定期予定やカレンダー連携に向く。
- Googleフォーム相当の直接入力: ファイルが苦手な人向けの最短ルート。
- URL投稿: 公式ページや公開PDFのURLを送ってもらい、自宅サーバー側で取得・変換する。

初期は「直接入力、PDF、Excel、Word、テキスト、CSV」までが現実的。画像OCRとURL取得は便利だが、誤読・権利・取得失敗があるため第2段階がよい。

## フォーム直接入力

フォーム直接入力は必要度が高い。ファイルを用意できない人でも送れるうえ、入力内容を最初から構造化できるため、変換ミスが少ない。

特に便利な機能:

- 前回入力の呼び出し: 同じ団体・同じ会場・同じ例会を再投稿しやすくする。
- パターン入力: 毎週、毎月第2火曜、最終金曜などの開催パターンを選べる。
- 複製して編集: 先月の予定をコピーして、日付や時間だけ直す。
- よく使う会場の保存: 会場名、住所、地図URL、備考を再利用する。
- 入力途中の一時保存: スマホで途中まで入力して後から続けられる。
- 送信前プレビュー: HTML表やカレンダー表示で、公開時にどう見えるか確認できる。

初期画面は「フォーム直接入力」と「ファイル投稿」を並べる。ただし、継続投稿する人にはフォーム直接入力を主導線にする。

前回呼び出しの保存単位:

- 投稿者単位
- 団体単位
- 例会単位
- 会場単位

保存するデータは、公開データそのものではなく「次回入力を楽にするための下書きテンプレート」として扱う。過去の誤情報をそのまま再投稿しないよう、送信前に日付・曜日・会場・公開範囲の確認を必須にする。

## 表示範囲と本体反映

断酒でGOGOアプリ上の表示範囲は翌月までとする。

例:

- 2026年7月中にGOGOで表示するデータ: 2026年8月末まで
- 2026年8月中にGOGOで表示するデータ: 2026年9月末まで

この制限は、断酒でGOGOアプリ内のHTML表、カレンダー印刷、検索結果に適用する。投稿自体は翌月以降の予定も受け付けてもよいが、GOGO上で利用者に見せる範囲は翌月末までに絞る。

断酒でGO本体へ反映するのは、確約分のみとする。GOGO内では「投稿済み」「確認中」「確約済み」「本体反映済み」のように状態を分け、断酒でGO本体の地図・検索・通常表示には確約済みのデータだけを渡す。

フォーム直接入力では、翌月を超える日付が入力された場合に「保存はできるが、GOGOでの表示は対象月になってから」と表示する。これにより、早めの予定提出は受け付けつつ、利用者向け画面には遠すぎる予定を出しすぎない。

## ドメイン方針

おすすめは、断酒でGOと同じルートドメイン配下で別アプリとして始めること。

例:

- 断酒でGO本体: `https://danshu.example/`
- 断酒でGO!!GO!!: `https://danshu.example/gogo/`
- 受付APIや管理画面: 必要なら `https://gogo.danshu.example/`

理由:

- 同じ例会データを扱うため、利用者には同じサービス群として見える。
- 将来統合しやすい。
- Android WebViewアプリも、本体用と投稿用で別々に作れる。
- フォルダを分ければ、同一ドメインでも別アプリとして動かせる。

完全に別ブランドや別ログイン基盤にするならサブドメインも有効。ただし最初は `/gogo/` の方が統合しやすい。

実際の候補:

- 断酒でGO本体: `dansyu-go.nukadokonokai.com`
- 断酒でGOGO: `dansyu-gogo.nukadokonokai.com`

既存ドメインが `dansyu-go` なので、GOGOも `dansyu-gogo` で揃えるのがよい。英字表記としては `danshu` も考えられるが、既存ドメインと混ぜると管理・案内・AndroidアプリURLで迷いやすいため、今は `dansyu` に統一する。

別案として `gogo.dansyu-go.nukadokonokai.com` もあるが、URLが長くなる。GOGOを別アプリとして見せるなら `dansyu-gogo.nukadokonokai.com` の方がわかりやすい。

## 実装方針

断酒でGOGOは、断酒でGO本体内に直接組み込まず、別アプリとして制作するのがよい。

理由:

- GOGOは投稿、下書き、確認中、確約済み、本体反映済みなどの作業状態を持つ。
- 断酒でGO本体は、確約済みの例会情報だけを見せる閲覧アプリとして保ちたい。
- 誰でも投稿できる入口を本体に混ぜると、未確認情報と確約情報の境界が曖昧になる。
- ファイル投稿、直接入力、前回呼び出し、印刷、カレンダー表示など、GOGO独自の画面が多い。
- 別アプリにしておくと、Androidアプリも本体用と投稿用に分けやすい。

ただし、完全に別世界にはしない。データ形式、会場ID、例会ID、確約済みデータの出力形式は断酒でGO本体と揃える。将来統合したい場合は、本体の中に `/gogo/` を取り込む、または管理・投稿機能として接続できるようにする。

## 機能整理

断酒でGOGOの機能は、大きく5つに分ける。

1. 入力する
2. 整理する
3. 確認する
4. 見る・印刷する
5. 断酒でGO本体へ渡す

### 入力する

- フォーム直接入力
- PDF投稿
- Excel投稿
- Word投稿
- テキスト投稿
- CSV投稿
- 前回入力の呼び出し
- パターン入力
- 複製して編集
- 入力途中の一時保存

### 整理する

- 投稿データの一覧
- 投稿者別の管理
- 団体別の管理
- 例会別の管理
- 会場別の管理
- 重複候補の検出
- 既存データとの差分表示

### 確認する

- 投稿済み
- 確認中
- 確約済み
- 本体反映済み
- 差し戻し
- 修正依頼
- 管理者メモ

### 見る・印刷する

- HTML表で見る
- カレンダー形式で見る
- 選んだ例会だけ一覧化する
- 選んだ例会だけ印刷する
- カレンダー印刷
- GOGO上の表示範囲は翌月末まで

### 断酒でGO本体へ渡す

- 確約済みデータだけを出力する
- 会場ID、例会ID、日付データを本体形式に合わせる
- 1日1回のバッチ反映
- 有料会員は即時反映ルートを検討する

## 試作で最初に作る範囲

最初の試作は、以下に絞る。

- フォーム直接入力
- 前回入力の呼び出し風UI
- 投稿一覧
- 状態管理: 投稿済み、確認中、確約済み、本体反映済み
- HTML表表示
- カレンダー表示
- 印刷用表示
- 確約済みだけを本体連携用JSONとして出す想定の画面

ファイル投稿、OCR、Word/PDF/Excel変換、自動差分検出、有料会員の即時反映は第2段階に回す。

## データ処理方針

- 一次情報の保管と変換は自宅サーバーで行う。
- 変換後の公開データをGitHubへ反映する。
- GitHub Pagesは閲覧用の静的公開先として使う。
- 原則は1日1回のバッチ反映。
- 有料会員は即時プッシュ反映できる別ルートを用意する。

安全のため、投稿データをそのまま公開データに混ぜず、以下の段階を分ける:

1. raw submissions: 投稿された元データ
2. extracted fields: AI/OCR等で抽出された候補データ
3. review state: 自動判定または人間確認
4. normalized data: 断酒でGO本体が使える例会・会場・日付データ
5. public JSON / HTML: GitHub Pagesへ出す公開成果物

## Android方針

同じドメイン内でフォルダ分けしても、Androidアプリは2つに分けられる。

- 断酒でGOアプリ: 本体URLを開く
- 断酒でGO!!GO!!アプリ: `/gogo/` を開く

Android側ではアプリID、アプリ名、アイコン、起動URLを変えれば別アプリとして配布できる。

## 変更点

- 新規プロジェクト用の初期設計メモを作成した。
- GO!!GO!!は別アプリとして始めつつ、同一ドメイン配下 `/gogo/` で将来統合しやすくする方針を記録した。
- 入力形式の追加候補として CSV、画像、ICS、直接入力、URL投稿を追加した。
- 投稿データを raw / extracted / review / normalized / public に分ける方針を記録した。
- フォーム直接入力を重要な初期機能として追記した。
- 前回入力の呼び出し、パターン入力、複製編集、一時保存、プレビュー確認を候補機能として追加した。
- 断酒でGOGO上の表示範囲は翌月まで、断酒でGO本体は確約分のみ表示する方針へ修正した。
- 断酒でGOGOは別アプリとして制作し、データ形式を断酒でGO本体に合わせる推奨方針を追加した。
- 断酒でGOGOの機能を「入力、整理、確認、見る・印刷、本体連携」に分類した。
- 試作で最初に作る範囲をフォーム直接入力中心に絞った。
