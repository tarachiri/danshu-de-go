# 2026-07-05 断かもLINE問い合わせ配送路の実装

## 背景

断酒でGO!!には、例会情報の修正・訂正・間違いなどを申告できるLINE問い合わせ入口がある。
一方で、505団体ページの「LINEで問い合わせる」ボタンから送られる最初の文面は
「○○のページから問い合わせ」であり、既存のエスカレーション条件に
「問い合わせ」が含まれていなかった。

そのため、問い合わせ入口は存在していても、内容がまじまじさんまで届かない可能性があった。

## 実装内容

対象: `gen:/Users/mini2014/danshu-chat/main.py`

- `ESCALATION_KEYWORDS` に `問い合わせ` / `問合せ` / `お問合せ` / `連絡` / `相談` / `訂正` を追加
- 問い合わせ検知後、同じLINEユーザーからの続きのメッセージを60分間管理者通知対象にする
- 管理者通知文に時刻・新規/続き・UserID・内容を入れるように整形
- `line_push_admin()` が成功/失敗を返し、失敗時はログに理由を残すように補強
- ユーザーへの受付返信を「詳しい内容があれば、このまま続けて送ってください」に変更

## 検証

- `python3 -m py_compile` で文法確認済み
- gen上で `/usr/local/bin/python3 -m py_compile main.py.inquiry-new` 実行済み
- `main.py.bak-20260705-line-inquiry-delivery` として反映前バックアップ作成済み
- `launchctl stop/start com.danshu.uvicorn` で再起動済み
- `curl http://127.0.0.1:8000/health` が `{"status":"ok","venues_loaded":1215}` を返すことを確認
- `line_push_admin()` を直接呼び出し、管理者Pushが `push_admin_ok=True` で成功することを確認

## 注意点

問い合わせ継続状態はFastAPIプロセスのメモリ上に保持しているため、再起動すると消える。
ただし、最初の問い合わせ文面自体に「問い合わせ」が含まれるため、通常の入口からの新規問い合わせは再起動後も検知される。
