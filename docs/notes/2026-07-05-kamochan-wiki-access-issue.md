# 課題メモ：かもちゃん（Claude.ai/API）からのWikiアクセス

## 背景・関連課題

tyo上にMkDocs Material + Dockerで開発Wiki（danshu-de-go / danshu-tools統合）を
構築する作業を別途進行中（別ファイル: wiki構築の指示書参照）。
Wikiは当初「tyoローカルのみ・外部非公開」の方針で設計。

追加要望として「かもちゃん（このClaude.aiチャット、またはLINEボット醸ちゃん）が
http://でWikiにアクセスできるようにしたい」という話が出た。

## 根本原因（構造的な制約）

かもちゃん（claude.aiのこのチャット）は Anthropic のクラウド上で動作しており、
まじまじさんのローカルネットワーク（tyo等）には物理的に到達できない。
これはDNSやポート開放やDockerの設定の問題ではなく、
「私がどこから実行されているか」という根本的な制約。

- web_fetchツールは「まじまじさんが提示したURLを、その都度1回読みにいく」動作のみ。
  常時巡回・自発的アクセスはできない。
- 「ローカルネットワーク内でのみ公開」と「かもちゃんがアクセスできる」は、
  かもちゃんがクラウド上にいる限り両立しない。

## 検討した選択肢

| 案 | 内容 | 評価 |
|---|---|---|
| URLに認証トークンを埋め込む | `http://tyo-ip:port/?token=xxxx` | 非推奨。アクセスログ・ブラウザ履歴・リファラーヘッダー経由で漏洩しやすい。却下。 |
| tyoにフォルダ作ってHTTP静的配信 | nginxやpython http.serverでdocsを直接公開 | mdが素のテキストのままでHTML変換されず、検索・ナビゲーションなし。目的（万能マニュアル）に対し根本解決にならない。 |
| Docker + MkDocs Material（構築済み方針） | mdをHTML変換、検索・目次付き | Wiki自体の使い勝手としては正解。ただし外部アクセス問題は別途解決要。 |
| Cloudflare Tunnel + Cloudflare Access | 既存cloudflaredインフラ（api.nukadokonokai.com等）に相乗り。Service Tokenで限定アクセス | 根本解決の候補。ただし「かもちゃんが常時読む」はweb_fetchの1回性のため、実現するのは「まじまじさんがURLを貼ってその都度読む」用途まで。LINEボット（醸ちゃん）側での常時参照は別の仕組み（ぬかちゃんのRAGと同種）が必要。 |
