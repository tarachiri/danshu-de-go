# 2026-07-12 docs/notes機能別目次作成メモ

## 今回の課題

`docs/notes/` にフロントエンド、断かも、GOGO、データ生成、Cloudflare、複数AI運用などの作業記録が日付順に蓄積され、機能から関連記録を横断的に探しにくくなっていた。

## 変更内容

- `docs/index.md` を新規作成した。
- 既存の`docs/notes/*.md` 32件を確認し、機能別に分類した。
- フロントエンド、地図UI、日程・会場、ニュース、LINE/Webチャット、GOGO、会員・団体、Cloudflare、インフラ、複数AI運用に整理した。
- 複数機能にまたがる記録は、それぞれの機能から辿れるよう重複掲載した。
- 今後の記録追加時に目次も更新する運用ルールを記載した。

## 確認内容

- `docs/index.md` 内のMarkdownリンク先がすべて存在することを検査する。
- 今回の記録を含む`docs/notes/*.md` がすべて目次に掲載されていることを検査する。
- 既存の作業記錱は変更していない。

## 解決した課題

日付やファイル名を知らなくても、機能から関連する設計、実装、障害、検証記録へ辿れる入り口を作成した。

## 今後の運用

新しい作業記録を追加するときは、`docs/index.md` の該当機能にリンクと一行要約を追加する。
