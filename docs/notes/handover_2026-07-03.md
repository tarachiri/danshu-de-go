## プロジェクト概要

**断酒でGO!!**（dansyu-go.nukadokonokai.com）
全国47都道府県・1,390以上の会場をカバーする断酒例会マップサービス。全日本断酒連盟に加盟する断酒会の例会を第一優先で収集（AAはスコープ外・利用者への言及不要）。コアコンセプトは「今日いける例会が30秒でわかる」。データ品質方針：正確な情報を量より優先。

### リポジトリ構成（恒久ルール）
- **Main**：https://github.com/tarachiri/danshu-de-go （HTML/CSS/JavaScript/Node.js＝フロントエンド）
- **tools**：https://github.com/tarachiri/danshu-tools （Pythonスクリプト保管庫）
- 役割は厳格分離。指示書は必ずどちらが対象か明記する
- **分離理由（2026-07-03確認）**：過去にgenerate_map_v6.pyのcron自動push（毎日5時）と人間の手動コミットが同一mainブランチ上で衝突した経緯があり、それを避けるため2リポジトリに分離した

## 検討中・未着手のテーマ

### venues.json / schedule.json 一元化（検討済み、未実施）

**背景：**
地図タブ（app.js）用のvenues.json（会場ごとにmeetings[]をネストした構造）と、日程タブ（schedule.js）用のschedule.json（1meeting=1レコードのフラット構造）が別々に存在し、内容が重複している。

**調査結果（generate_map_v6.py確認済み）：**
schedule.jsonは、venues.json生成後にそのentries変数を`meetings[]`でループしてフラット展開しているだけで、**DBへの独自クエリを持たず、venues.jsonから100%機械的に導出可能**なことが確定している。

```python
# generate_map_v6.py内、該当箇所
for v in entries:
    for m in v.get("meetings", []):
        if not m.get("next_date"):
            continue
        schedule_entries.append({...})
```

**一元化の方針案：**
| 箇所 | 変更内容 | 難易度 |
|---|---|---|
| generate_map_v6.py（danshu-tools） | schedule.json生成部分（約29行）を削除 | 簡単 |
| app.js（danshu-de-go） | 変更不要 | なし |
| schedule.js（danshu-de-go） | venues.jsonをfetchし、同じフラット化ロジックをJS側に移植 | 簡単 |
| gen-main.schedule-work.py（gen、断かもRAG） | `fetch_schedule()`・`schedule_data`・`_build_schedule_entry()`を削除し、venues_dataベースの共通関数に統一 | 中 |

**副次的メリット：** cronの自動コミット対象がvenues.json単独になり、コンフリクト発生源が単純化される。

**ステータス：** 骨子は固まっているが、指示書（指示書2）はまだ作成していない。venue登録共通化（指示書1）を先行させる合意のもと、一旦保留中。

### venue（施設）とmeeting（例会）の設計思想確認

「施設は固定的（変動性が低い）、例会は流動的（会場変更・休止・追加情報・不測の事態）」という設計思想を確認。既存のDB構造（venuesテーブル／meetingsテーブルをvenue_idで1対多に紐づけ）は、この思想と整合していることを確認済み。

**venue昇格の基準（promote_raw_meetings.py確認済み）：**
raw_meetings→meetings昇格の判定は、施設名＋都道府県の文字列マッチのみ（完全一致→前方一致8文字以上）で、**緯度経度は一切使用されていない**。

**新規venue登録処理の分散状況：**
`INSERT INTO venues`を持つスクリプトは、cron現役稼働分に限ると以下5つ（venue登録共通化タスクで既に`register_venue()`に統合済み）：
- shizuoka_build_venues.py、osaka_shi_scraper.py、hokusetsu_scraper.py、osaka_rengo_scraper.py、sakaishi_scraper.py
