# フッターがマップ画面で隠れる問題／日程・お知らせタブでの表示

*作成日: 2026-07-04*

## 課題

画面下部のフッター（`©2026 ぬか床の会 | 開発ノート`表示）が、
マップ画面・日程タブ・お知らせタブのいずれでも実質的に表示されて
いなかった。要望は「マップ画面では非表示のままでよいが、日程タブと
お知らせタブでは画面下にフッターを表示したい」というタブごとの
出し分け。

利用者が例会の日程やお知らせを見ている最中に、運営情報（開発ノートへの
導線）へアクセスできない状態が続いていたため、断酒会員がサービス全体の
運営状況を確認する手段が実質失われていたことになる。

## 根本原因

`body`が `display: flex; flex-direction: column; height: 100dvh;
overflow: hidden;` という画面いっぱいの固定レイアウトになっており、
スクロールが一切できない。`#footer`はDOM順で一番下にある通常の
ブロック要素だったため、`#bottom-nav`（`position: fixed;
z-index: 1000`）の裏に隠れる形になり、どのタブを開いても実質見えない
状態だった。加えて `switchTab()` 関数にはそもそも `#footer` の表示・
非表示を切り替えるロジックが存在しなかった。

## 対応内容

1. **`switchTab()`（app.js）にフッター出し分けロジックを追加**
   マップタブ選択時は `#footer` を非表示・`#map-attribution`
   （Leaflet著作権表示バー、別件で新設済み）を表示。それ以外の
   タブ（日程・お知らせ・かもちゃん）では逆に `#footer` を表示、
   `#map-attribution` を非表示にする。

2. **`#footer`をposition: fixedに変更（style.css）**
   `overflow: hidden`固定レイアウトの中では通常フローの要素が
   スクロール不可能な領域に埋もれてしまうため、`#map-attribution`と
   同じ設計思想で `position: fixed; bottom: 60px;`
   （ボトムナビ直上）に固定。高さ16pxの帯として、はみ出た文字は
   `text-overflow: ellipsis` で省略表示にした。

3. **日程・お知らせタブのボトム余白を再計算**
   `#schedule-view` / `#news-view` の `padding-bottom` を
   60px→76px（ボトムナビ60px＋フッター16px）に変更。
   `#news`（index.htmlインラインstyle）の `max-height` も
   `100vh - 120px` → `100vh - 136px` に修正し、フッターの
   16px分を追加で差し引いた。

対象ファイル: `app.js`, `index.html`, `style.css`
（いずれもdanshu-de-goリポジトリ）
