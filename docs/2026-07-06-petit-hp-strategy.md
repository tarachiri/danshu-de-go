# プチHP（org-XXX）運用方針の検討 - 東北6県から着手

対象リポジトリ：danshu-de-go（本メモ）／danshu-tools（generate_chiiki_pages_v3.pyの改修時）

断酒でGO!!は、アルコール依存症からの回復・断酒支援を目指す全日本断酒連盟加盟の断酒会・例会情報を案内するサービス。本メモは、その入口の一つである「プチHP（org-XXX、団体別ページ）」の運用方針を検討したもの。

## 背景：なぜ今プチHPを見直すのか

きっかけは「断酒でGOのポップアップ内、公式サイトボタンがある団体とない団体がある。見づらい・繋がらない団体をどう案内するか」という相談だった。調査の結果、以下が判明した。

1. プチHP（`chiiki/{block}/{pref}/org-XXX/`）は全505件、`generate_chiiki_pages_v3.py` が `zendanren_organizations`（全断連公式冊子2026年版をOCRした団体マスタ）を無条件で全件生成している
2. 除外条件は `is_suspended`（活動休止）・`is_umbrella`（連合会等の上部組織）の2つのみ。「公式サイトが生きているか」を見るフィルタは存在しない
3. 東京の実例（東京多摩断酒連合会＝org-519）で、公式サイト（`tama-danshurengou.jpn.org`）を持つ団体にもプチHPが生成されていることを確認
4. まじまじさんの肌感覚では、**約半数**の団体が「公式サイトの維持管理ができずソーバーネットに相乗り」「ネット社会についていけず公式サイト自体たどり着かない」状態

## プチHPの本来の目的（再定義）

プチHPは「公式サイトの有無」で機械的に分岐させるものではなく、**ネットから孤立している断酒会を拾い上げるためのセーフティネット**。対象になるのは：

- 公式サイト自体が存在しない団体
- 公式サイトはあるが更新されていない（スマホ非対応・情報が古い・連絡先が古い・検索でも辿り着けない）

逆に、iCalやPDFを定期更新している「生きている」公式サイトは、そのまま公式サイトとして案内してよい（今後スマホ非対応の場合は別途「公式情報スマホビュー」の検討対象）。

## 東北6県の実地調査（2026-07-06時点）

venues.json（会場・例会データ）の状態：

| 県 | venue総数 | meetings有 | fallbackのみ | official_url有 |
|---|---|---|---|---|
| 青森県 | 4 | 2 | 2 | 4 |
| 岩手県 | 0 | 0 | 0 | 0 |
| 宮城県 | 0 | 0 | 0 | 0 |
| 秋田県 | 0 | 0 | 0 | 0 |
| 山形県 | 0 | 0 | 0 | 0 |
| 福島県 | 0 | 0 | 0 | 0 |

**発見1**：岩手・宮城・秋田・山形・福島の5県は、venue（会場・例会情報）が1件も登録されていない。一方でプチHP（団体マスタ由来）は各県2〜7件存在する（青森2, 秋田5, 岩手6, 山形6, 福島7, 宮城1）。つまり「団体の存在は把握できているが、例会情報の収集が届いていない」状態。

**発見2**：青森県の4venue（id: 701〜704）は、`official_url`に `gallery/mtg_g/....jpeg`（画像パス）や `meeting.html?mtg_id=1947&date=...`（相対パス＋クエリ）が入っており、`fallback_meeting_name` に「コロナ禍におけるアルコールの存在」「HALT」等、通常の例会名と異なる文言が見られた。ソーバーネット（全断連公認NPO）由来のデータと推測される。ソーバーネットは全断連公認のため、このデータ自体は使用して問題ない。ただしURL形式が壊れている（相対パス・画像パス）ため、そのままofficial_urlとして案内するのは要修正。

## 決定した方向性

1. **東北6県から着手する。** プチHPに載せる情報は、まじまじさんが個別に収集した基本情報を軸に構成を固めていく。汎用化できる部分は随時、他県へも展開する。
2. **個人情報の扱い**：責任者名・個人電話番号などは、本人確認が取れるまでプチHP上に非表示とする。代わりに、LINE公式アカウント（断かも）とWebチャット（かも）で質問に答える形にする。実現方法（確認フローの設計等）は今後の課題として保留。
3. プチHP要否の判定基準（「例会情報の有無・鮮度を代理指標にする」）は、Wiki `phase1-organizations-design`（organizations新設・未実装）の設計と接続できる見込み。`organizations`テーブルに `site_status` 相当の概念を持たせられないか、Phase 1設計側で今後検討する。
